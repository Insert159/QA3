# Тест-дизайн и тест-анализ

Проект по применению техник тест-дизайна на практике

## О проекте

В проекте изучены и применены основные техники тест-дизайна: классы эквивалентности, анализ граничных значений, таблицы переходов и состояний, попарное тестирование, предугадывание ошибок. Также рассмотрены тестовые туры и методы расчета тестового покрытия.

## Что выполнено

✨ **Задание 1. Классы эквивалентности**

Определены классы эквивалентности для поля ввода логина на кредитном ресурсе. Для каждого класса указаны значения для тестирования.

🎯 **Задание 2. Граничные значения**

Проанализирован онлайн-калькулятор кредитной ставки. Определены граничные значения с учетом условий: ставка 20% до 10000 руб., 15% от 10000 до 50000 руб., 10% от 50000 руб. Максимальная сумма 100000 руб., шаг 1000 руб.

🔄 **Задание 3. Таблица переходов и состояний**

Составлена таблица переходов и состояний для страницы авторизации applicant.21-school.ru. Схема оформлена визуально в виде диаграммы.

⚡ **Задание 4. Попарное тестирование**

Для анкеты кредитования (возраст, задолженности, тип трудовых отношений, действующие кредиты) подсчитано количество комбинаций и составлены уникальные пары с применением техники pairwise.

🧠 **Задание 5. Тестовые туры**

Изучена тема тестовых туров. На основе чек-листа из проекта QA2 предложены 2 варианта тестовых туров с описанием сути, фокуса и действий тестировщика. Использована нейросеть для генерации идей.

📊 **Задание 6. Матрица тестового покрытия**

На основе дополненного чек-листа из QA2 составлена матрица тестового покрытия. Функционал разделен на три категории: Критический, Важный, Вспомогательный с обоснованием каждой категории.

## Технологии и темы

| Техника | Описание |
|---------|----------|
| Классы эквивалентности | Разбиение входных данных на группы с одинаковым поведением системы |
| Анализ граничных значений | Проверка значений на границах классов эквивалентности |
| Таблицы переходов и состояний | Моделирование поведения системы в разных состояниях |
| Попарное тестирование | Сокращение комбинаций параметров без потери качества |
| Предугадывание ошибок | Использование опыта для поиска вероятных дефектов |
| Тестовые туры | Исследовательское тестирование с разных углов зрения |
